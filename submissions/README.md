# 제출물(Submissions)

이 폴더에 5개 과제의 산출물을 직접 작성한다. `../verify.sh` 가 아래 파일의
존재와 최소 분량을 점검한다. (정답 채점이 아니라 "완료 기준" 점검)

| 파일 | 과제 | 핵심 내용 |
|---|---|---|
| `01-triage.md`    | Task 1 | 증상 요약, 영향 범위, 위반/위협받는 SLO, 1차 보고(#incident-war-room 톤) |
| `02-rca-report.md`| Task 2 | 진단 타임라인, 사용한 지표/PromQL, 근본 원인 가설과 **근거** |
| `05-rfc.md`       | Task 5 | 재발 방지 아키텍처 개선안(RFC) + 외부 유사 장애 포스트모템 1건 비교 |

> Task 3(완화)·Task 4(Slack 연동)는 **클러스터 상태**로 검증되므로 별도 파일이 없다.
> (HPA/limit 변경, Alertmanager 리시버·룰 추가 등 실제 적용 여부를 verify.sh 가 본다.)

작성 팁: 추측을 적지 말고 **무엇을 보고 그렇게 판단했는지**(지표 캡처/쿼리/이벤트)를
함께 남겨라. 이게 실제 포스트모템의 핵심이다.
